How resolution works
  • The decoder runs fully offline; WebAssembly is loaded from this folder (no inline blob).
  • Bare part refs like {80} resolve as {itemTypeId:80}.
  • Typed refs like {246:22} keep their own family id.
  • When multiple dataset rows share the same id, the resolver prefers the current manufacturer and item type.
  • Unresolved typed refs can show optional hints from data/part_ref_meta.js (loaded beside this page) when that id is documented there.
This version keeps YAML support but restores the actual offline decode path.
